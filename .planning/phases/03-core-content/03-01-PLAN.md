<task type="auto" tdd="true">
  <name>Task 2: Rewrite comment API routes</name>
  <files>app/api/comments/route.ts, app/api/comments/[id]/route.ts, app/api/comments/route.test.ts</files>
  <behavior>
    - POST /api/comments without auth returns 401
    - POST /api/comments with auth stores author_id from JWT (not from body)
    - POST /api/comments with invalid body returns 400 with Zod field errors
    - POST /api/comments with parent_id creates reply linked to parent
    - POST /api/comments with send_to_captain=true stores the flag
    - GET /api/comments?videoId=X returns top-level comments with reply counts, sorted oldest first
    - GET /api/comments?parentId=X returns replies for that parent
    - GET /api/comments?captainOnly=true requires auth
    - PATCH /api/comments/[id] without auth returns 401
    - PATCH /api/comments/[id] by non-owner returns 403
    - PATCH /api/comments/[id] by owner updates text, sets is_edited=true, updates updated_at
    - DELETE /api/comments/[id] by non-owner returns 403
    - DELETE /api/comments/[id] by owner deletes comment
  </behavior>
  <action>
1. Rewrite `app/api/comments/route.ts`:
   - GET: Keep existing query logic (videoId, sessionId, captainOnly, parentId, type=qa filtering). Add Zod validation on query params using CommentQuerySchema. Keep reply_count RPC logic.
   - POST: REQUIRE auth via getTokenPayload(req). Validate body with CreateCommentSchema.safeParse(). Insert with `author_id: payload.userId` — do NOT accept author_name or author_id from request body. Derive author_name from payload.userName for backward compat column. Return 201 with created comment.

2. Create `app/api/comments/[id]/route.ts`:
   - PATCH: Require auth. Validate body with EditCommentSchema. Fetch existing comment, verify `author_id === payload.userId` (or payload.role === 'captain' for moderation). Update comment_text, set is_edited=true, updated_at=now. Return updated comment.
   - DELETE: Require auth. Verify ownership (author_id === payload.userId OR captain). Delete comment. If deleting a parent with replies, also delete all replies (cascade). Return 204.

3. Write tests in `app/api/comments/route.test.ts` covering the behaviors listed above. Mock supabase and getTokenPayload per established test patterns.
  </action>
