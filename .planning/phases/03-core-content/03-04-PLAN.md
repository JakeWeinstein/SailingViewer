<!-- Comment API contracts from Plan 01 -->
```
POST /api/comments — auth required, body: { video_id, session_id, timestamp_seconds, comment_text, send_to_captain, parent_id }
GET /api/comments?videoId=X — returns comments with reply_count, sorted oldest first
GET /api/comments?parentId=X — returns replies for parent
PATCH /api/comments/[id] — auth required, body: { comment_text }, sets is_edited
DELETE /api/comments/[id] — auth required, ownership check
```

<!-- Session API contracts from Plan 03 -->
```
GET /api/sessions — returns sessions with session_videos, each video has comment_count + flagged_count
POST /api/sessions/[id] { action: 'close' } — captain close + carry forward
POST /api/sessions/[id] { action: 'add-video', youtube_url } — add video to session
```

  <name>Task 1: VideoWatchView comment experience</name>
  <files>components/VideoWatchView.tsx</files>
  <action>
Rewrite the comment section of VideoWatchView.tsx to implement the full comment experience. Keep the existing video player and chapter sidebar unchanged — only modify the comment area.

**Comment composer:**
1. Auto-fill timestamp from `player.getCurrentTime()` when the comment textarea receives focus — only if player state is PLAYING(1) or PAUSED(2). Display as formatted time (e.g., "2:34") in a removable badge next to the textarea. User can click X to clear or manually edit.
2. "Flag for captain review" checkbox/toggle below the textarea. Maps to `send_to_captain: true` on POST.
3. Submit calls POST `/api/comments` with auth. Include `video_id`, `session_id`, `timestamp_seconds`, `comment_text`, `send_to_captain`. On success, append the new comment to the local list.
4. For replies: clicking "Reply" on a comment opens a nested composer under that comment. Reply composer has same timestamp + flag behavior. POST with `parent_id` set.

**Comment list:**
1. Fetch comments on mount via GET `/api/comments?videoId=X`. Sorted oldest first (API handles sort).
2. Each comment shows: author name, time ago, timestamp badge (if present), comment text, reply count badge, edit/delete buttons (if own comment).
3. Clicking a timestamp badge calls `player.seekTo(seconds, true)` to seek the video.
4. Reply count badge shows e.g., "3 replies". Clicking it fetches GET `/api/comments?parentId=X` and expands replies below the parent. Click again to collapse. Single-level only (no sub-replies).
5. Edit: clicking edit shows an inline textarea pre-filled with comment text. Save calls PATCH `/api/comments/[id]`. On success, update local state, show "edited" indicator.
6. Delete: clicking delete shows confirmation, then calls DELETE `/api/comments/[id]`. On success, remove from local list.

**30-second polling:**
1. While the comment section is visible (mounted), set up a 30-second interval that re-fetches comments.
2. Use the append-only pattern from RESEARCH.md: compute a Set of existing IDs, filter fresh results to only new ones, append without disrupting scroll position.
3. Cleanup interval on unmount. Handle StrictMode double-invocation via cleanup function.
4. Silent failure on network errors — don't show error toasts for polling failures.

**Styling:**
- Comments: `border-b border-gray-100 py-3` for each comment
- Timestamp badge: `bg-blue-100 text-blue-700 px-1.5 py-0.5 rounded text-xs font-mono cursor-pointer hover:bg-blue-200`
- "edited" indicator: `text-gray-400 text-xs italic` next to timestamp
- Flag checkbox: small checkbox with label "Flag for captain"
- Reply toggle: `text-sm text-gray-500 hover:text-gray-700 cursor-pointer`

The component needs to know the current user's ID to show edit/delete buttons. Accept `userId` and `userRole` as props (passed down from the page component).
