  <name>Task 3: Reference Manager UI — tags + chapters</name>
  <files>components/ReferenceManager.tsx</files>
  <action>
Extend `ReferenceManager.tsx` with tag management and inline chapter adding:

**Tag filtering UI (top of reference library view):**
1. Fetch all existing tags on mount via GET `/api/reference-videos?allTags=true`.
2. Render filter chips at the top. Each chip is a tag name — click to toggle active. Active chips have a filled/highlighted style. Multiple active = AND filter.
3. When filter chips are active, pass `?tags=x,y` to the reference videos fetch. Videos not matching are hidden.

**Tag editing per video:**
1. In the video detail/edit view, show current tags as removable chips.
2. Tag input field below chips with autocomplete dropdown: as user types, show matching existing tags (filtered by input, case-insensitive). Pressing Enter or clicking a suggestion adds the tag.
3. New tags (not in autocomplete) are also accepted — freeform system.
4. On tag change, PATCH `/api/reference-videos/[id]` with updated tags array.

**Inline chapter adding (extends existing chapter UI):**
1. The existing chapter list shows chapters under a parent video. Add a "+ Add Chapter" button.
2. On click: show inline form with title input and timestamp input (auto-filled from current context if available, or manual entry in MM:SS format).
3. On submit: POST `/api/reference-videos` with `parent_video_id`, `title`, `start_seconds` (parsed from timestamp input).
4. Any logged-in user can add/edit/delete chapters — trust-based, per locked decision.

Use Tailwind for styling. Tag chips: `px-2 py-0.5 rounded-full text-sm bg-blue-100 text-blue-800` (active) / `bg-gray-100 text-gray-600` (inactive). Autocomplete dropdown: absolute positioned below input, `bg-white shadow-lg rounded border max-h-40 overflow-y-auto`.
