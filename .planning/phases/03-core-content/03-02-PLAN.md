**ArticleEditor.tsx — Full redesign:**

1. Import dnd-kit: `DndContext`, `closestCenter` from `@dnd-kit/core`; `SortableContext`, `verticalListSortingStrategy`, `useSortable`, `arrayMove` from `@dnd-kit/sortable`; `CSS` from `@dnd-kit/utilities`.

2. Internal state uses `ArticleBlockWithId` = `ArticleBlock & { _id: string }`. Generate `_id` via `crypto.randomUUID()` on block creation. Strip `_id` before saving to API.

3. Block insertion buttons at top and between blocks: "+ Text", "+ Video", "+ Image", "+ Clip". Each creates an empty block of that type with a stable `_id`.

4. Each block wrapped in a `SortableBlock` component using `useSortable({ id: block._id })`. Render a drag handle icon (GripVertical from lucide-react) with `{...listeners}` and `{...attributes}`. Include delete button per block.

5. Block editors by type:
   - **Text**: `<textarea>` for markdown input + live preview toggle using `react-markdown`. Textarea should be auto-resizing.
   - **Video**: YouTube video URL/ID input field + optional title + optional caption. Parse with `extractYouTubeInfo()` from lib/types.ts. Show thumbnail preview.
   - **Image**: URL input + optional alt text + optional caption. Validate URL client-side with `new URL()`. Show preview with `<img onError>` fallback.
   - **Clip**: YouTube video URL/ID input + start time (MM:SS format) + optional end time (MM:SS format) + optional title + optional caption. Parse times with `parseTimestamp()`. Show YouTube thumbnail preview.

6. `handleDragEnd(event)`: Use `arrayMove` to reorder blocks array. Find indices by matching `_id` against `active.id` and `over.id`.

7. Save handler: Strip `_id` from all blocks before POSTing/PATCHing to `/api/articles` or `/api/articles/[id]`. Existing save logic for title, is_published, etc. should be preserved.

8. Keep existing arrow buttons as keyboard-accessible fallback alongside drag handles.

**ArticleViewer.tsx — Extend for new block types:**

1. Keep existing text block rendering (ReactMarkdown).
2. Keep existing video block rendering (YouTube iframe).
3. Add **image block**: `<figure>` with `<img src={block.url} alt={block.alt}>` and `<figcaption>` for caption. Add `onError` handler to show placeholder text "Image could not be loaded".
4. Add **clip block**: YouTube iframe with `?start=X&end=Y` URL parameters. Show caption below. Format timestamp range as "X:XX - Y:YY" label.
5. Handle unknown block types gracefully (skip, don't crash).
