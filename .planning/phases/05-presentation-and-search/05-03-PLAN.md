**1. Create `components/GlobalSearchBar.tsx` (client component):**
- Form with search icon (Search from lucide-react) and text input.
- On submit (Enter key or form submit), navigate to `/search?q=${encodeURIComponent(query.trim())}` using `router.push`.
- Do NOT fetch on every keypress (Research anti-pattern). Only navigate on submit.
- Style: rounded-full input with pl-9 for icon, border-gray-200, focus ring blue-500. Expand width on focus (w-48 -> w-64 via transition-all). On mobile, collapse to icon-only and expand on tap.
- Accept optional `className` prop for layout integration.
- If the current route is `/search`, pre-fill the input from `useSearchParams().get('q')`.

**2. Create `app/search/page.tsx` (server component wrapper):**
- This is a public page (no auth required, /search in PUBLIC_PATHS).
- Wrap `<SearchResults>` in `<Suspense fallback={<div>Loading...</div>}>` to satisfy useSearchParams requirement (Research Pitfall 1).
- Basic page layout: centered max-w-4xl container with padding, title "Search Results".

**3. Create `components/SearchResults.tsx` (client component, ~150 lines):**

**Data fetching:**
- Read `q` from `useSearchParams()`.
- On mount or when `q` changes, fetch `GET /api/search?q=${q}&limit=40`.
- State: `results` (array), `loading`, `error`, `expandedSections` (Set of types that are expanded).

**Results grouping:**
- Group results by `type` into four sections: Videos, Comments, Articles, Q&A.
- Section order: Videos -> Comments -> Articles -> Q&A (fixed order).
- Each section has a header with icon and count badge.
- By default show top 5 results per section. "Show more" button expands to show all.
- If a section has 0 results, hide it entirely.
- If all sections empty, show "No results found for '{q}'" message.

**Result cards (per type):**
- **Video**: Film icon, title, snippet, timeAgo. Click navigates to `/?session=${url_hint}&video=${id}` (or however the home page video routing works — check app/page.tsx for existing video deep-link pattern).
- **Comment**: MessageSquare icon, author name (title field), snippet with timestamp badge if snippet starts with "[" (timestamp prefix from RPC). Click navigates to video player at that timestamp: determine the URL from url_hint (video_id).
- **Article**: FileText icon, article title, snippet. Click navigates to `/learn/${id}`.
- **Q&A**: HelpCircle icon, author name, snippet. Click navigates to `/?tab=qa&post=${id}` (or however Q&A deep-linking works — check app/page.tsx).

**Scroll position preservation (Research Pattern 4):**
- Before navigating to a result, save `window.scrollY` to `sessionStorage` with key `scroll:/search?q=${q}`.
- On mount, check sessionStorage for saved scroll position. If found, `window.scrollTo(0, saved)` and remove the key.

**Navigation function:**
```typescript
function navigateToResult(url: string) {
  if (typeof window !== 'undefined') {
    sessionStorage.setItem(`scroll:${window.location.pathname}${window.location.search}`, String(window.scrollY))
  }
  router.push(url)
}
```

**Empty/loading states:**
- Loading: skeleton cards (3 per section placeholder).
- Error: red error message with retry button.
- Empty query (no q param): show search prompt "Enter a search term to find videos, comments, articles, and Q&A posts."

**4. Integrate GlobalSearchBar into `app/page.tsx`:**

- Position it to be visible and accessible but not dominate the layout.

**5. Integrate GlobalSearchBar into `components/DashboardView.tsx`:**

- Visible to all dashboard roles (captain, contributor, viewer).

**Important:** Do NOT add GlobalSearchBar to app/layout.tsx — keep it per-page as recommended by Research (Option 2), since the home page and dashboard have different header layouts.
