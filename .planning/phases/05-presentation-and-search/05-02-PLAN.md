**2. Create `components/PresentationMode.tsx` (client component, ~200+ lines):**

**Data fetching:**
- State: `items` (queue items), `activeItemId` (from searchParams), `showArchived` (toggle), `referencePanelOpen`, `selectedSession`.
- On session select or mount, fetch `GET /api/comments?sessionId=X&captainOnly=true` to get all flagged items + Q&A.
- Also fetch Q&A with no session: `GET /api/comments?type=qa&captainOnly=true` for items with `session_id IS NULL` (general Q&A).
- Separate into active (`is_reviewed=false`) and archived (`is_reviewed=true`) using client-side filter. Sort active by `sort_order`.
- Group items by `author_name` into collapsible sections, sorted alphabetically by author. Within each author group, items sorted by `sort_order`.

**Layout (split-pane):**
- Left pane (~350px, scrollable): `<PresentationQueue>` component
- Right pane (flex-1): Active item detail area
  - If no item selected: empty state with instructions
  - If item selected: Show the comment/Q&A text, author info (avatar, name, timeAgo), and below it:
    - If flagged comment (has video_id): embed YouTube player using the video's youtube_id from session_videos. Show timestamp badge if timestamp_seconds exists. Player should seek to timestamp on load.
    - If Q&A post (no video_id): show the Q&A text, any youtube_attachment embed.
  - Below video/text: inline reply field (textarea + send button). On submit, POST /api/comments with parent_id=activeItemId, session_id, comment_text. Fire-and-forget notification.
  - "Mark as Reviewed" button (green checkmark). Calls PATCH /api/comments/[id] with {is_reviewed: true}. On success, remove item from active list optimistically.

**Toggle archived view:**
- Tab or toggle button at top of queue: "Active" | "Archived"
- Archived view shows is_reviewed=true items, each with a "Restore" button that calls PATCH with {is_reviewed: false}.

**URL-addressability:**
- Use `useSearchParams` to read `?item=uuid`. On item click, `router.replace` to update param (not push, so back exits presentation).
- On mount, if `?item=uuid` exists, set that as activeItemId.

**Keyboard shortcuts:**
- Use `useEffect` with `keydown` listener on `document`.
- ArrowDown/ArrowRight: select next item in flattened queue order (across groups).
- ArrowUp/ArrowLeft: select previous item.
- R key: mark current item as reviewed (only when no input/textarea focused).
- Escape: navigate back to `/dashboard` (router.push).
- Guard: skip shortcuts if `event.target` is input, textarea, or contenteditable.

**Session picker:**
- Dropdown at top of presentation mode to select which session's queue to view.
- Default to the active session.

**Exit button:**
- Top-right "Exit" button or X icon that navigates back to `/dashboard`.

**3. Create `components/PresentationQueue.tsx` (client component, ~120 lines):**

**Sortable queue with dnd-kit:**
- Receive items (grouped by author), activeItemId, onSelectItem, onReorder callbacks as props.
- Wrap in `DndContext` with `closestCenter`, `PointerSensor` (distance: 8), `TouchSensor` (delay: 250, tolerance: 5).
- Use `SortableContext` with `verticalListSortingStrategy`.
- Each author group: collapsible section header (author name + avatar + item count). Default expanded.
- Each item: `useSortable` wrapper. Show item type icon (MessageSquare for comment, HelpCircle for Q&A from lucide-react), truncated text (first 80 chars), timestamp badge if applicable.
- Active item highlighted with blue-500 ring/background.
- On `handleDragEnd`: compute new order with `arrayMove`, call `onReorder` with the full reordered array.
- The `onReorder` callback in PresentationMode: optimistically update local state, then PATCH /api/comments/reorder with { session_id, order }.
- Drag handle icon (GripVertical from lucide-react) on left of each item.

**Visual distinction per item type:**
- Flagged comments: MessageSquare icon, show "[timestamp]" badge in blue.
- Q&A posts: HelpCircle icon, show "Q&A" badge in purple.
