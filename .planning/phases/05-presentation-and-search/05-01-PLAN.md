  <files>supabase-migration-phase5.sql</files>
  <action>
Create `supabase-migration-phase5.sql` with:

1. **Add review lifecycle columns to comments:**
```sql
ALTER TABLE comments
  ADD COLUMN IF NOT EXISTS is_reviewed BOOLEAN NOT NULL DEFAULT false,
  ADD COLUMN IF NOT EXISTS reviewed_at TIMESTAMPTZ,
  ADD COLUMN IF NOT EXISTS sort_order INTEGER;
```

2. **Create partial index for fast queue fetch:**
```sql
CREATE INDEX IF NOT EXISTS idx_comments_queue
  ON comments(session_id, is_reviewed, sort_order)
  WHERE send_to_captain = true AND parent_id IS NULL;
```

3. **Initialize sort_order for existing flagged items (per-session chronological sequence):**
```sql
UPDATE comments
SET sort_order = sub.rn
FROM (
  SELECT id,
         ROW_NUMBER() OVER (PARTITION BY session_id ORDER BY created_at) - 1 AS rn
  FROM comments
  WHERE send_to_captain = true AND parent_id IS NULL
) sub
WHERE comments.id = sub.id;
```

4. **Create full-text search RPC function `search_all`:**
- Parameters: `search_query text`, `result_limit int DEFAULT 20`
- Returns TABLE(id uuid, type text, title text, snippet text, url_hint text, rank real, created_at timestamptz)
- UNION ALL across four sources:
  - **session_videos**: search on `title` field, type='video', url_hint=session_id::text
  - **comments** (with video_id, top-level only): search on `comment_text`, type='comment', title=author display_name (JOIN users), url_hint=COALESCE(video_id::text, session_id::text). Include `timestamp_seconds` in snippet as prefix like "[2:34] comment text..."
  - **articles** (is_published=true only): search on `title || content extracted from blocks`, type='article', url_hint=id::text. For snippet, extract text block content using `jsonb_array_elements` to get only `content` fields from text blocks, not raw JSON.
  - **comments** (Q&A: video_id IS NULL, parent_id IS NULL): search on `comment_text`, type='qa', title=author display_name, url_hint=id::text
- Use `websearch_to_tsquery('english', search_query)` for query parsing
- Use `ts_rank(to_tsvector('english', ...), q)` for ranking
- ORDER BY rank DESC, LIMIT result_limit
- For article snippet extraction, use a lateral join or subquery: `SELECT string_agg(elem->>'content', ' ') FROM jsonb_array_elements(a.blocks) elem WHERE elem->>'type' = 'text'` to get clean text without JSON noise (addresses Research Pitfall 6).

5. **Run migration via Supabase MCP** if available, otherwise document for manual execution.

**1. Update `lib/schemas/comments.ts`:**
- Extend `EditCommentSchema` to also accept optional `is_reviewed` (boolean) and `sort_order` (number, nonneg). Use `z.union` or make a separate `ReviewCommentSchema` — prefer a separate schema to keep edit and review concerns separate:
```typescript
export const ReviewCommentSchema = z.object({
  is_reviewed: z.boolean(),
})
export const ReorderSchema = z.object({
  session_id: z.string().uuid(),
  order: z.array(z.object({
    id: z.string().uuid(),
    sort_order: z.number().int().nonnegative(),
  })).min(1),
})
export const SearchQuerySchema = z.object({
  q: z.string().min(1).max(200).trim(),
  limit: z.coerce.number().int().min(1).max(100).default(20),
})
```

**2. Extend `app/api/comments/[id]/route.ts` PATCH handler:**
- After the existing edit logic, add a new code path: if body contains `is_reviewed` (boolean), parse with `ReviewCommentSchema`.
- Captain-only check: `if (payload.role !== 'captain') return 403`
- Update: `{ is_reviewed: body.is_reviewed, reviewed_at: body.is_reviewed ? new Date().toISOString() : null }`
- Keep the existing edit path (comment_text) working as-is.
- Strategy: Try `EditCommentSchema` first; if it fails, try `ReviewCommentSchema`; if both fail, return 400. This avoids breaking the existing edit flow.

**3. Create `app/api/comments/reorder/route.ts`:**
- PATCH endpoint, captain-only (`getTokenPayload` + role check).
- Parse body with `ReorderSchema`.
- Use `Promise.all` to update each comment's `sort_order` where `id` matches and `session_id` matches (double-check session ownership).
- Return `{ ok: true }`.

**4. Create `app/api/search/route.ts`:**
- GET endpoint, public (no auth required).
- Parse `searchParams.get('q')` and optional `limit` with `SearchQuerySchema`.
- Call `supabase.rpc('search_all', { search_query: q, result_limit: limit })`.
- Return the results array as JSON.
- If `data` is null or error, return appropriate error response.

**5. Update `middleware.ts`:**
- Add `'/search'` to `PUBLIC_PATHS` array so the search page is accessible without authentication.
