  <name>Task 1: User management and invite code API routes</name>
  <files>app/api/users/route.ts, app/api/users/[id]/route.ts, app/api/users/[id]/reset-password/route.ts, app/api/settings/invite-code/route.ts</files>
  <action>
**A. Create `app/api/users/route.ts`:**
- GET handler (captain-only): getTokenPayload, check role==='captain', return 403 if not.
- Query users table: select id, username, display_name, role, is_active, is_seed, last_login_at, created_at. Order by created_at desc.
- For comment counts, use a separate count query or join: `SELECT u.*, (SELECT COUNT(*) FROM comments WHERE author_id = u.id) as comment_count FROM users u ORDER BY created_at DESC`. Execute via supabase.rpc or two queries.
- Return 200 with users array (exclude password_hash).

- PATCH handler (self-service profile): getTokenPayload. Parse body with UpdateProfileSchema.safeParse. On failure return 400.
- If newPassword provided: fetch user from DB, bcrypt.compare currentPassword, if wrong return 403.
- Hash newPassword if provided. Update display_name and/or password_hash. Return 200 with updated user.

**B. Create `app/api/users/[id]/route.ts`:**
- PATCH handler (captain-only): getTokenPayload, check captain role.
- Parse body with UpdateRoleSchema.safeParse. On failure return 400.
- Fetch target user. If target user is_seed=true AND new role !== 'captain', return 403 `{ error: 'Cannot demote the seed captain' }`.
- A captain cannot change their own role to non-captain (prevent self-lockout) unless another captain exists. Simplification: just prevent demoting is_seed.
- Update user role. Return 200.

- DELETE handler (captain-only): getTokenPayload, check captain role.
- Fetch target user. If target is_seed=true, return 403 `{ error: 'Cannot delete the seed captain' }`.
- Cannot delete self. Return 403 if attempting.
- Delete user (CASCADE handles comments, articles). Return 200.

- Also add a PATCH for deactivation: accept `{ is_active: boolean }` when the body has that shape. Use a combined schema:
```typescript
const UserUpdateSchema = z.union([
  UpdateRoleSchema,
  z.object({ is_active: z.boolean() }),
])
```

**C. Create `app/api/users/[id]/reset-password/route.ts`:**
- POST handler (captain-only): getTokenPayload, check captain.
- Parse body with ResetPasswordSchema.safeParse. On failure return 400.
- Hash temporaryPassword with bcrypt.
- Update password_hash and set must_change_password=true.
- Return 200 `{ success: true }`.

**D. Create `app/api/settings/invite-code/route.ts`:**
- GET handler (captain-only): read invite_code from app_config. Return 200 `{ inviteCode }`.
- POST handler (captain-only): generate new UUID, update app_config invite_code value and updated_at. Return 200 `{ inviteCode: newCode }`.
