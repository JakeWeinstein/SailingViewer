**A. Create `supabase-schema-v2.sql`** — the complete new schema for a fresh Supabase project. This is the canonical DDL (not a migration diff). Include all tables:

1. `users` — id UUID PK, username TEXT UNIQUE NOT NULL, display_name TEXT NOT NULL, password_hash TEXT NOT NULL, role TEXT NOT NULL DEFAULT 'viewer' CHECK (role IN ('captain','contributor','viewer')), is_active BOOLEAN NOT NULL DEFAULT true, is_seed BOOLEAN NOT NULL DEFAULT false, must_change_password BOOLEAN NOT NULL DEFAULT false, last_login_at TIMESTAMPTZ, created_at TIMESTAMPTZ DEFAULT NOW(). Index on username.

2. `sessions` — id UUID PK, label TEXT NOT NULL, is_active BOOLEAN DEFAULT false, created_at TIMESTAMPTZ DEFAULT NOW(). NO videos JSONB column.

3. `session_videos` — id UUID PK, session_id UUID NOT NULL REFERENCES sessions(id) ON DELETE CASCADE, drive_file_id TEXT NOT NULL, title TEXT NOT NULL, position INTEGER NOT NULL DEFAULT 0, note TEXT, note_timestamp INTEGER, created_at TIMESTAMPTZ DEFAULT NOW(). Index on session_id.

4. `comments` — id UUID PK, session_id UUID REFERENCES sessions(id) ON DELETE CASCADE, video_id UUID REFERENCES session_videos(id) ON DELETE CASCADE, author_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE, timestamp_seconds INTEGER, comment_text TEXT NOT NULL CHECK (char_length(comment_text) <= 2000), send_to_captain BOOLEAN DEFAULT false, parent_id UUID REFERENCES comments(id) ON DELETE CASCADE, created_at TIMESTAMPTZ DEFAULT NOW(). Indexes on video_id, session_id, parent_id, author_id.

5. `reference_folders` — keep existing structure exactly (id, name, description, parent_id self-ref, sort_order, created_at).

6. `reference_videos` — keep existing structure exactly (id, title, type, video_ref, note, note_timestamp, folder_id FK, parent_video_id self-ref, start_seconds, sort_order, created_at).

7. `articles` — keep existing structure but ensure author_id references users(id) ON DELETE SET NULL. Keep author_name for display caching. Add folder_id UUID column if not present.

8. `app_config` — key TEXT PK, value TEXT NOT NULL, updated_at TIMESTAMPTZ DEFAULT NOW().

At the bottom, seed the invite code: `INSERT INTO app_config (key, value) VALUES ('invite_code', gen_random_uuid()::text);`

**B. Create `scripts/migrate.ts`** — a Node.js script that migrates from the old schema to the new one. This script:
1. Imports supabase client from `../lib/supabase` (without server-only, see note below)
2. Reads CAPTAIN_USERNAME and CAPTAIN_PASSWORD from env (or defaults: 'captain' / env CAPTAIN_PASSWORD)
3. Hashes the captain password with bcryptjs
4. Runs SQL via `supabase.rpc('exec_sql', ...)` or uses raw SQL through the Supabase management API. Since we can't run raw SQL through the JS client easily, structure this as a `.sql` migration file instead: create `scripts/migrate.sql` that:
   - Creates new tables (session_videos, app_config) if not exist
   - Alters users table: add is_active, is_seed, must_change_password, last_login_at columns; alter role CHECK to include 'viewer'; change default to 'viewer'
   - Truncates comments, sessions (CASCADE), users
   - Drops videos column from sessions
   - Seeds invite code into app_config
5. Create `scripts/migrate.ts` as a helper that hashes the captain password with bcrypt and prints the INSERT statement for the seed captain (since bcrypt hashing can't happen in raw SQL without pgcrypto being enabled).

The migrate.ts script should:
```typescript
import bcrypt from 'bcryptjs'

async function main() {
  const username = process.env.CAPTAIN_USERNAME || 'captain'
  const displayName = process.env.CAPTAIN_DISPLAY_NAME || 'Captain'
  const password = process.env.CAPTAIN_PASSWORD
  if (!password) { console.error('Set CAPTAIN_PASSWORD env var'); process.exit(1) }
  const hash = await bcrypt.hash(password, 12)
  console.log(`-- Run this after migrate.sql:`)
  console.log(`INSERT INTO users (username, display_name, password_hash, role, is_active, is_seed)`)
  console.log(`VALUES ('${username}', '${displayName}', '${hash}', 'captain', true, true)`)
  console.log(`ON CONFLICT (username) DO NOTHING;`)
}
main()
```
