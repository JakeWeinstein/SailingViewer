  <files>supabase-migration-phase4.sql, lib/schemas/notifications.ts, lib/schemas/bookmarks.ts, lib/schemas/comments.ts, lib/mention-utils.ts, lib/__tests__/mention-utils.test.ts</files>
  <action>
1. Create `supabase-migration-phase4.sql` with:
   - CREATE TABLE notifications (id UUID PK DEFAULT gen_random_uuid(), user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE, type TEXT NOT NULL CHECK (type IN ('mention', 'reply', 'captain_response')), source_id UUID NOT NULL, is_read BOOLEAN NOT NULL DEFAULT false, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW())
   - CREATE INDEX idx_notifications_user_id ON notifications(user_id, is_read, created_at DESC)
   - CREATE TABLE bookmarks (id UUID PK DEFAULT gen_random_uuid(), user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE, video_id TEXT NOT NULL, session_id UUID REFERENCES sessions(id) ON DELETE SET NULL, timestamp_seconds INTEGER NOT NULL CHECK (timestamp_seconds >= 0), video_title TEXT, created_at TIMESTAMPTZ NOT NULL DEFAULT NOW())
   - CREATE INDEX idx_bookmarks_user_id ON bookmarks(user_id, created_at DESC)
   - CREATE UNIQUE INDEX idx_bookmarks_unique ON bookmarks(user_id, video_id, timestamp_seconds)
   - ALTER TABLE comments ADD COLUMN IF NOT EXISTS youtube_attachment TEXT (nullable YouTube video ID)

2. Create `lib/schemas/notifications.ts`:
   - MarkReadSchema: z.object({ id: z.string().uuid().optional(), markAll: z.boolean().optional() }).refine(data => data.id || data.markAll, { message: 'Provide id or markAll' })

3. Update `lib/schemas/comments.ts`:
   - Add youtube_attachment: z.string().max(20).optional() to CreateCommentSchema

4. Create `lib/schemas/bookmarks.ts`:
   - CreateBookmarkSchema: z.object({ video_id: z.string().min(1), session_id: z.string().uuid().optional(), timestamp_seconds: z.number().nonnegative().int(), video_title: z.string().max(200).optional() })

5. Create `lib/mention-utils.ts` (NOT server-only — shared between server routes and client rendering):
   - parseMentions(text: string): Array<{ type: 'text' | 'mention'; value: string }> — splits text by @username tokens using /@([a-zA-Z0-9_]+)/g regex. For 'mention' type, value includes the @ prefix.
   - createMentionNotifications(commentId: string, commentText: string, authorId: string, supabase: SupabaseClient): Promise<void> — parses @username patterns, looks up user IDs via supabase.from('users').select('id, username').in('username', usernames), filters out self-mentions (authorId), bulk inserts notification rows with type='mention'.
   - createReplyNotification(commentId: string, parentId: string, authorId: string, supabase: SupabaseClient): Promise<void> — fetches parent comment's author_id, if different from authorId inserts a single notification with type='reply'.
   - Note: createMentionNotifications and createReplyNotification import supabase at call site — they take it as param to keep the module importable from client (for parseMentions).

6. Create `lib/__tests__/mention-utils.test.ts` — behavioral tests for parseMentions:
   - Test: empty string returns single empty text segment
   - Test: text with no mentions returns single text segment
   - Test: "@alice" returns single mention segment with value "@alice"
   - Test: "hey @alice check this" returns [text "hey ", mention "@alice", text " check this"]
   - Test: multiple mentions "@alice and @bob" returns correct segments
   - Test: "@alice@bob" (no space) — should handle edge case (implementation decides)
   - Test: special chars "email@test.com" — should or should not match (implementation decides, but test documents behavior)
   - Test: "@Alice_123" with underscores and numbers returns mention segment
   - These tests run with `npx vitest run lib/__tests__/mention-utils.test.ts` and validate the core parsing logic used across all mention features.

1. Create `app/api/notifications/route.ts`:
   - GET: auth required (getTokenPayload). If searchParams has countOnly=true, return { unread: count } via supabase.from('notifications').select('id', { count: 'exact', head: true }).eq('user_id', payload.userId).eq('is_read', false). Otherwise return last 30 notifications ordered by created_at DESC with a join to comments to build deep-link data: for each notification, join source_id to comments table to get video_id, session_id, parent_id, comment_text (first 100 chars as preview), author_name. Construct link field: if video_id exists -> `/?video=${source.video_id}&session=${source.session_id}&comment=${source.id}`, if no video_id -> `/?view=qa&post=${source.parent_id || source.id}`.
   - PATCH: auth required. Parse body with MarkReadSchema. If id provided, update single notification set is_read=true WHERE id=id AND user_id=payload.userId. If markAll=true, update all WHERE user_id=payload.userId AND is_read=false. Return { success: true }.

2. Create `app/api/bookmarks/route.ts`:
   - GET: auth required. Return supabase.from('bookmarks').select('*').eq('user_id', payload.userId).order('created_at', { ascending: false }).
   - POST: auth required. Parse body with CreateBookmarkSchema. Insert with user_id=payload.userId. On duplicate (Supabase error code 23505 or error message containing 'unique'), return 409 { error: 'Bookmark already exists' }. Otherwise return 201 with data.

3. Create `app/api/bookmarks/[id]/route.ts`:
   - DELETE: auth required. Fetch bookmark by id, verify user_id matches payload.userId (ownership check). If not owner, return 403. Delete and return 204 (empty NextResponse with status 204).

   - After successful insert: call createMentionNotifications(data.id, comment_text, payload.userId, supabase) — import from '@/lib/mention-utils'.
   - After successful insert: if parent_id is provided, call createReplyNotification(data.id, parent_id, payload.userId, supabase) — import from '@/lib/mention-utils'.
